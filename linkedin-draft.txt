Claude Code에 "하네스"를 씌웠더니, 아이디어가 배포까지 한 세션에 끝났다

어제 랜딩 페이지를 하나 만들었습니다.
아이디어 정리부터 Vercel 배포까지, 한 세션에.

솔직히 처음엔 Claude Code를 "똑똑한 자동완성" 정도로 썼습니다.
"이 컴포넌트 만들어줘" → 코드 복붙 → 안 되면 에러 던지기.
결과물은 나왔지만, 항상 어딘가 빠져 있었습니다.

이번엔 달랐습니다. superpowers라는 스킬 하네스를 붙였거든요.

달라진 것:

하네스 없이 쓸 때는 제가 모든 걸 기억해야 했습니다.
"다음 뭐 하지?" "이거 빠뜨린 거 아닌가?" "이 코드 맞나?"
AI가 코드를 짜주는데, 정작 저는 PM + 리뷰어 + QA를 혼자 하고 있었어요.

하네스를 쓰니 흐름이 구조화됐습니다.
브레인스토밍 → 비주얼 목업 리뷰 → 디자인 스펙 → 스펙 리뷰 에이전트 → 구현 플랜 → 플랜 리뷰 에이전트 → 코드 구현 → 배포.

특히 리뷰 에이전트가 결정적이었습니다.
구현 플랜이 Tailwind v3 문법으로 작성됐는데, 실제 설치된 건 v4.
환경변수가 클라이언트에 노출되는 보안 이슈까지.
생성한 AI가 못 잡는 걸, 검증하는 AI가 잡았습니다.

이게 핵심이었어요.
AI 코딩의 병목은 "생성 속도"가 아니라 "검증 부재"였습니다.

전: 빠르게 만들고, 느리게 고치고, 빠뜨린 거 나중에 발견
후: 느리게 검증하고, 한 번에 제대로 만들고, 빠뜨린 게 없음

결과적으로 한 세션에 6개 섹션 랜딩 페이지가 기획부터 배포까지 끝났습니다.
프레임워크 삽질 없이.

AI를 잘 쓴다는 건, 더 많이 시키는 게 아니라
실수를 잡아주는 구조를 만드는 것이었습니다.
